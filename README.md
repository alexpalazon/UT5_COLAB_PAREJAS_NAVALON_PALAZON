## Objetivo de la práctica: 

El objetivo de esta práctica es trabajar de forma colaborativa utilizando Git y GitHub,
simulando un entorno real de desarrollo en equipo. Cada integrante crea su propia rama,
realiza cambios de manera independiente y posteriormente integra su trabajo en la rama
principal mediante Pull Requests.

## Comandos Git usados

- `git clone`: clona un repositorio remoto en el equipo local.
- `git status`: muestra el estado actual del repositorio.
- `git remote -v`: muestra la conexión con el repositorio remoto.
- `git branch`: lista las ramas existentes.
- `git checkout -b`: crea una nueva rama y cambia a ella.
- `git add`: añade archivos al área de preparación.
- `git commit`: guarda los cambios realizados en el historial del repositorio.
- `git push`: sube los cambios al repositorio remoto.
- `git pull`: actualiza el repositorio local con los cambios del remoto.
- `git merge`: integra los cambios de una rama en otra.
